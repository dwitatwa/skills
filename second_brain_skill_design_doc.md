# Three-Layer Second-Brain Concept

## Overview
This document describes a general-purpose second-brain concept built around two core ideas:
- a three-layer knowledge architecture
- the Zettelkasten method

The purpose is to define a simple conceptual model for how a future Skill could maintain knowledge over time.

Rather than treating notes as a flat collection of documents, this model separates raw material, durable knowledge, and maintenance rules into distinct layers.

## The Three Layers

### 1. Sources Layer
The sources layer contains raw material in its original form.

Examples:
- PDFs
- articles
- meeting notes
- transcripts
- web clippings
- bookmarks
- personal notes
- highlights
- screenshots
- exported documents

The main role of this layer is preservation.

Principles:
- source materials remain unchanged
- this layer acts as the evidence layer
- later notes can refer back to this layer for provenance and context
- the layer is for capture, not interpretation

This layer is important because understanding may change over time, but the original material should remain stable.

### 2. Knowledge Layer
The knowledge layer is the actual second brain.

This layer contains the notes that represent understanding, interpretation, and reusable ideas.

In this concept, the knowledge layer should follow the Zettelkasten method rather than a wiki-first structure.

That means the main unit is not the long topic page, but the atomic note.

The knowledge layer should contain:
- literature notes
- permanent notes
- structure notes
- open-question notes
- source notes
- index or hub notes

Principles:
- notes should stay small and reusable
- each note should usually contain one idea
- notes should be linkable across contexts
- notes should grow into a network rather than a rigid hierarchy
- uncertainty can remain visible instead of being forced into a final answer

This layer is where information becomes connected knowledge.

### 3. Rules Layer
The rules layer defines how the system stays structured and maintainable.

This layer does not primarily store knowledge content. Instead, it defines how the knowledge layer should be created and maintained.

It can define:
- note types
- note templates
- metadata rules
- linking rules
- provenance rules
- update rules
- uncertainty rules
- maintenance rules

Principles:
- the layer should keep the system consistent
- it should guide when to create a new note versus update an existing one
- it should guide how notes link to one another
- it should preserve clarity between evidence, interpretation, and unresolved questions

This layer is what prevents the second brain from becoming an unstructured note pile.

## How the Layers Work Together
The three layers serve different purposes and should remain conceptually separate.

- the sources layer preserves raw input
- the knowledge layer turns raw input into linked ideas
- the rules layer governs how that transformation happens and how the graph stays coherent

A useful way to think about the model is:
- sources layer = capture
- knowledge layer = understanding
- rules layer = maintenance

This separation is important because each layer solves a different problem.

## Zettelkasten Method in This Model
The Zettelkasten method is the organizing principle of the knowledge layer.

Instead of creating large summary pages, the system builds a network of smaller notes that can connect across many contexts.

The emphasis is on:
- atomic ideas
- meaningful links
- gradual accumulation of insight
- structure emerging from connections

This makes the knowledge layer more flexible than a traditional notebook or wiki structure.

## Core Zettelkasten Note Types

### Literature Notes
Literature notes capture ideas that come directly from a source.

They are usually:
- concise
- source-linked
- close to the original material
- useful for extracting candidate permanent notes

### Permanent Notes
Permanent notes represent durable ideas in a reusable form.

They are usually:
- written in the user’s own words
- focused on one idea
- meant to stay useful beyond the original source
- designed to connect with many other notes

### Structure Notes
Structure notes help organize groups of notes.

They are usually:
- maps of related notes
- entry points into a topic cluster
- evolving guides rather than fixed taxonomies

### Open-Question Notes
Open-question notes preserve uncertainty.

They are useful for:
- unresolved tensions
- contradictions
- research directions
- incomplete thoughts that should remain open

### Source Notes
Source notes provide a lightweight reference point for source material.

They help connect the sources layer to literature notes and other downstream notes.

## Why Zettelkasten Fits the Three-Layer Model
The Zettelkasten method fits especially well because it does not assume that knowledge begins as a polished summary.

Instead, it allows knowledge to emerge gradually through smaller linked units.

That aligns naturally with the three-layer model:
- the sources layer preserves what was captured
- the knowledge layer turns captured material into atomic linked notes
- the rules layer ensures those notes stay coherent over time

A wiki-first system tends to center on topic pages. A Zettelkasten-centered system instead centers on linked ideas.

For a second brain, that distinction matters because reusable thought often grows from relationships between small notes rather than from static summary pages.

## Conceptual Flow
The overall flow of the model is simple:
1. raw material enters the sources layer
2. key ideas are extracted into literature notes
3. durable ideas are expressed as permanent notes
4. notes are linked to related notes
5. structure notes help organize clusters as they emerge
6. the rules layer keeps the whole system coherent

This flow is not a one-time process. It is an ongoing cycle of capture, distillation, linking, and maintenance.

## Summary
This concept defines a second brain around two foundations:
- a three-layer architecture
- a Zettelkasten-style knowledge system

The three layers keep raw material, knowledge, and maintenance logic separate.
The Zettelkasten method gives the knowledge layer its form by emphasizing atomic notes, links, and gradual synthesis.

Together, these ideas create a model for a second brain that is more maintainable, more flexible, and more useful over time than a flat collection of notes or a purely summary-based system.
