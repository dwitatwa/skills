# Zettelkasten Rules

Use this reference when changing note templates or deciding how the knowledge layer should behave.

## Core behavior

- Treat the Zettelkasten as a personal thinking and writing tool, not as a generic document dump.
- Build a web of thoughts. The primary unit is the note, not the long topic page.
- Favor connection over collection. Every durable note should be easy to link from and link to.

## Note construction

- Give each note a stable unique ID. For digital notes, default to a timestamp such as `YYYYMMDDHHMM`.
- Keep one main idea per note. Atomicity is a guiding direction, not a rigid law, but large multi-idea notes should be split when possible.
- Write the body in the user's own words. Quotes can support a note, but they should not replace the note's interpretation.
- Keep provenance visible. If a note comes from a source, record the source reference. If it is an original thought, leave the provenance section empty rather than inventing a source.

## Linking

- Connect new notes to existing notes as soon as a real relationship exists.
- State why a link exists. Link context is part of the knowledge, not optional decoration.
- Use structure notes as entry points into clusters of notes. They are maps and hubs, not rigid top-down categories.

## Anti-patterns

- Do not turn the knowledge layer into a flat inbox of unsorted documents.
- Do not rely on tags or folder hierarchy alone to express relationships.
- Do not use large wiki-style pages as the default note format.
