Summary

This commit strengthens the exam submission lifecycle by enforcing strict registration checks and improving the robustness of the admin analysis view. Key improvements include:

  • Server-side validation of participant registration prior to submission.
  • Idempotent handling of duplicate submissions to prevent inconsistent results.
  • Introduction of "Orphan Results" in the admin dashboard to surface data for participants whose registration records have been modified or removed.
  • Client-side session recovery and automatic state clearing on authorization failure.
Note: The commit message was "fix issue". Technical intent was derived from control flow analysis.

Behavioral Changes

1. Resilient Submission Recovery

When a submission is rejected (403/404), the client now performs a graceful exit. It preserves a local backup of the answers before clearing the session, preventing data loss while stopping the infinite retry loop.

2. Orphan Result Visibility

Admins can now see results from participants who are no longer registered in the selected schedule. These are flagged with an "Unregistered Result" badge and include metadata about their actual registrations.
